# 🏠 Home Assistant Supervised - ULTIMATE INSTALLER

[![Bash Version](https://img.shields.io/badge/Bash-4%2B-blue)](https://www.gnu.org/software/bash/)
[![License](https://img.shields.io/badge/License-MIT-green)](LICENSE)

Этот скрипт превращает любую ТВ-приставку (X96, Tanix, HK1 на Amlogic/Allwinner), одноплатный компьютер (Raspberry Pi, Odroid) или обычный мини-ПК в полноценный, защищенный и отказоустойчивый сервер Home Assistant Supervised.

Установка HA Supervised на нестандартное железо — сложный процесс, связанный с проблемами в ОС, сетью, AppArmor и обрывами SSH. Этот скрипт решает все проблемы автоматически: от подмены os-release и патча загрузчика до защиты сессии SSH и авто-отката сети.

## ⚠️ Важное предупреждение

Скрипт вносит изменения в сетевой стек (переключает на NetworkManager), ядро (AppArmor) и файрвол. **Обязательно имейте физический доступ (монитор/клавиатура) или UART-подключение к устройству на случай обрыва сети по SSH.** Скрипт имеет систему отката, но 100% гарантию на всех китайских TV-боксах дать невозможно.

---

## ✨ Ключевые особенности

- **🛡️ Умная установка с откатами:** Если шаг установки падает (например, ломается сеть), скрипт автоматически откатывает изменения.
- **🔄 Автопродолжение после ребута:** Если AppArmor требует перезагрузки, скрипт создает systemd-сервис, который продолжает установку с того же места после старта системы.
- **🌐 Бесшовная интеграция Tailscale:** Автоматическая установка, настройка UFW (открытие портов WireGuard и интерфейса tailscale0) и поддержка Auth Key.
- **📶 Надежный Wi-Fi:** Настройка через NetworkManager с принудительным **отключением энергосбережения** (Power Save off), что решает проблему отвала VPN и телеметрии на TV-боксах. Поддержка SSID со спецсимволами и пробелами.
- **🛡️ Безопасность из коробки:** Настройка UFW (включая правила для Docker), Fail2Ban, защита SSH.
- **💾 Продвинутые бэкапы:** Поддержка локальных снапшотов (через HA CLI), загрузка в облако (rclone) или на удаленный сервер (rsync).
- **🚑 Режим восстановления (Rescue):** Автоматическая диагностика и починка файловой системы, сети, Docker и Supervisor.
- **📊 Мониторинг:** Watchdog с экспоненциальной задержкой, термомониторинг, Prometheus метрики.

---

## 📋 Требования

- **ОС:** Armbian (Bookworm / Trixie) или чистый Debian (11/12/13).
- **Архитектура:** `aarch64` (ARM64), `x86_64`.
- **RAM:** Минимум 1 ГБ (рекомендуется 2 ГБ+).
- **Диск:** Минимум 10 ГБ свободного места.
- **Права:** Скрипт необходимо запускать от `root` (`sudo`).

## 🚀 Быстрый старт

Самый простой способ запустить установщик — использовать интерактивный мастер:

```bash
curl -fsSL https://raw.githubusercontent.com/iRespect777/HAS-tvbox/refs/heads/main/ha-installer/install.sh -o /tmp/install.sh
sudo bash /tmp/install.sh
```
Мастер проведет вас через выбор профиля, настройку сети, часового пояса и дополнительных компонентов.

🔬 Глубокое погружение (Как это работает)
1. Установка и подмена os-release
HA Supervised официально поддерживает только Debian. Armbian, Ubuntu или специфичные сборки TV-боксов не пройдут проверку инсталлятора.
Скрипт бэкапит оригинальный /etc/os-release, создает фейковый (Debian 12/13), устанавливает HA, а затем создает systemd drop-in для hassio-supervisor.service:

ExecStartPre: Подменяет os-release перед стартом Supervisor.
ExecStopPost: Возвращает оригинальный os-release при остановке.
Это позволяет HA работать корректно, не ломая систему.
2. Безопасное переключение сети (Network Rollback)
Установка NetworkManager поверх ifupdown по SSH — самый частый способ "убить" сервер.
Скрипт действует осторожно:

Делает бэкап /etc/network/interfaces и /etc/resolv.conf.
Останавливает networking и запускает NetworkManager.
Если вы подключены по SSH, скрипт дает 15 секунд паузы перед переключением.
Ждет появления IP. Если IP нет — автоматически откатывается на ifupdown и dhclient, спасая доступ.
3. AppArmor и авто-перезагрузка
На многих боксах AppArmor отключен в ядре. Скрипт патчит конфиг загрузчика (/boot/extlinux/extlinux.conf или /boot/armbianEnv.txt), добавляя apparmor=1 security=apparmor.
Поскольку для применения параметров ядра нужна перезагрузка, скрипт:

Создает systemd сервис ha-install-continue.
Этот сервис стартует после ребута и запускает скрипт с флагом --from-step apparmor.
Установка бесшовно продолжается. (Лимит перезагрузок — 3, чтобы избежать boot-loop).
4. Интеграция с Tailscale
UFW: Открывает tailscale0 для свободного доступа из VPN и 41641/udp для прямых P2P-соединений (WireGuard), что критично для минимизации задержек в Умном доме.
Wi-Fi Power Save: TV-боксы агрессивно усыпляют Wi-Fi чипы. Скрипт использует nmcli, чтобы надежно отключить 802-11-wireless.powersave по UUID соединения (работает даже с SSID, содержащими пробелы и спецсимволы).
5. Уведомления (Webhooks)
Утилита ha-notify "умная": она определяет тип сервиса по URL и отправляет запрос в правильном формате:

ntfy.sh: Отправляет plain text с заголовками.
Discord / Slack / Gotify: Формирует корректный JSON с экранированием спецсимволов.
Кастомные: Сначала пробует plain text, если сервер отвечает ошибкой — отправляет JSON.
6. Система бэкапов
Скрипт пытается использовать нативную утилиту ha (HA CLI) для создания полных снапшотов (сохраняет аддоны, БД, Zigbee2MQTT). Если утилита недоступна, он делает быстрый tar архив папки /config (исключая тяжелые БД и кэш TTS).
Поддерживается удаленная отправка бэкапов через rclone (Яндекс.Диск, Google Drive и др.) и rsync (по SSH).

🗑 Удаление
Для удаления используйте аргумент --uninstall. Скрипт предложит два режима:

Standard: Удаляет HA, Supervisor, OS-Agent, контейнеры и утилиты. Оставляет Docker, NetworkManager и системные пакеты нетронутыми. Идеально, если хотите переустановить HA.
Full: Полная зачистка. Удаляет Docker, восстанавливает оригинальную сеть (возвращает ifupdown), сбрасывает UFW, удаляет Tailscale, очищает fstab от оптимизаций. Система вернется к состоянию до установки.
🚑 Решение проблем
Скрипт завис или оборвался по SSH
Не паникуйте! Скрипт использует систему состояний (/var/lib/ha-installer/state). Просто переподключитесь по SSH и запустите скрипт снова. Он увидит файл состояния и продолжит установку с прерванного шага.

Home Assistant не открывается после установки
Первичная загрузка HA может занимать 10-15 минут (он скачивает образы Supervisor, DNS, CLI и т.д.). Проверьте статус контейнеров:

bash

sudo docker ps
Если проблема в сети или Docker, запустите авто-починку:

bash

sudo ha-install --rescue
Tailscale отваливается через несколько минут
Это признак того, что ваш Wi-Fi адаптер уходит в спящий режим (Power Save). Скрипт пытается отключить это, но если не получилось:

Узнайте UUID вашего Wi-Fi: nmcli con show
Отключите энергосбережение: sudo nmcli con modify "ИМЯ_СЕТИ" 802-11-wireless.powersave 2
Переподключитесь: sudo nmcli con up "ИМЯ_СЕТИ"
Не работает DNS внутри контейнеров (ошибка загрузки HACS)
Скрипт пытается исправить это, добавляя "dns": ["8.8.8.8", "1.1.1.1"] в /etc/docker/daemon.json. Если HACS всё равно не скачивается:

bash

docker restart homeassistant
📂 Структура файлов установщика
/var/lib/ha-installer/ — Рабочий каталог установщика.
state — Файл состояния (какие шаги выполнены).
config — Конфигурация установки (профиль, версии, опции).
secrets/ — Защищенные токены (Telegram, Tailscale Auth Key, Webhooks).
backup/ — Бэкапы системных файлов (os-release, interfaces, resolv.conf).
/var/backups/homeassistant/ — Каталог для локальных бэкапов HA (tar.gz).
/usr/local/bin/ha-* — Утилиты управления (watchdog, backup, notify).
/etc/cron.d/ha-tools — Расписание задач (Cron).
/var/log/ha_install_*.log — Подробные логи последней установки.
